================================================================================
FROZEN PREPROCESSING CONTRACT
================================================================================

CRITICAL DESIGN PRINCIPLE:
  Data leakage prevention through preprocessing versioning.

TRAINING PHASE:
  1. Fit imputation model on training data ONLY
  2. Fit StandardScaler on training data ONLY
  3. Lock parameters in CleansingState
  4. Save state as JSON checkpoint

TEST/HOLDOUT PHASE:
  1. Load frozen CleansingState
  2. Apply identical imputation & scaling
  3. Never refit any preprocessing model
  4. Validate against frozen parameters

ENFORCEMENT MECHANISMS:
  - DataCleaner.fit() → locks state, raises error if called twice
  - DataCleaner.apply() → requires prior fit(), uses frozen params
  - CleansingState.version → tracks preprocessing version
  - Changes to cleansing → require explicit version bump

BENEFITS:
  ✓ No data leakage from test into training
  ✓ Reproducible preprocessing across runs
  ✓ Audit trail of preprocessing choices
  ✓ Version-locked validation protocol

    --output-dir data/processed \
    --imputation mice \
    --split-strategy stratified_group_kfold \
    --verbose

PROGRAMMATIC:
  from researcher1_clinical import (
    CoMMpassIngester, DataCleaner, FeatureEngineer, DataSplitter
  )
  
  # Ingest
  raw_df = CoMMpassIngester("data/raw").ingest()
  
  # Cleanse
  cleaner = DataCleaner(imputation_strategy="mice")
  cleaner.fit(raw_df)
  cleaned_df, mask = cleaner.apply(raw_df)
  
  # Engineer
  engineer = FeatureEngineer()
  engineered_df = engineer.engineer(cleaned_df)
  
  # Split
  splitter = DataSplitter()
  train_df, val_df, test_df = splitter.split(engineered_df)
